# SSAFY 14기 AI 챌린지 - 이미지 기반 질의응답 모델 개발 AI 챌린지

#### 대회 개요

이미지 속 세상을 '읽고, 이해하고, 답하는' 인공지능, 바로 **VQA(Visual Question Answering) 모델**은 인간의 시각적 사고에 가장 가까운 형태의 AI 기술입니다. 이번 AI 챌린지는 이미지와 텍스트를 함께 이해하는 멀티모달 인공지능을 직접 개발하고 평가하는 대회로, 주어진 이미지와 자연어 질문을 입력받아 정답을 a, b, c, d 중 하나로 선택하는 VQA 모델을 학습하고 평가해야 합니다. 대회는 실제 데이터셋 기반으로 Kaggle 리더보드를 통해 실시간 점수를 확인하며, AI의 정확도를 겨루게 됩니다.

## 2-1. GPU 메모리 제약 및 확장 학습의 한계 (OOM)

### 접근방향
초기에는 “많이 학습시키면 모델이 똑똑해지지 않을까?”라는 단순한 생각으로 접근했습니다. 그래서 T4 GPU(약 14.74 GiB) 환경에서 학습 데이터셋을 200개에서 전체 데이터셋(약 4000장)으로 확장했습니다.

### 문제점
(당연하지만) **GPU 메모리 부족 오류**(`OutOfMemoryError: CUDA out of memory`)가 발생했습니다. 이는 모델의 활성화 값, 그래디언트, 옵티마이저 상태가 GPU 메모리 한계를 초과했기 때문입니다. 특히 **역전파(`.backward()`) 단계**에서 메모리 부족이 집중되었습니다.

### 해결 방안

- 이로 인해 2가지 해결 방안으로 접근했습니다. (1) 추가 학습을 위한 백업 코드 설정, (2) 메모리 최적화입니다.
- 메모리 최적화를 위해 해당 2가지 파라미터를 조정했습니다.
- **기울기 누적 (Gradient Accumulation, GRAD_ACCUM 확대:** 배치 크기 1을 유지하면서 GRAD_ACCUM 값을 4에서 8 또는 16으로 증가시켜, **물리적 메모리 사용량 증가 없이 유효 배치 크기 효과**를 얻었습니다.
- **LoRA Rank 조정:** LoRA를 8에서 4 등으로 낮춰 학습 가능한 파라미터 수를 줄여 메모리 부하를 경감하는 방안을 고려했습니다. 이는 T4 GPU에서만 미봉책으로 사용했고, GPU 배포 이후에는 LoRA값을 다시 조정했습니다.

## 2-2. 데이터 활용 효율성 - 데이터 증강

### 접근법
- 데이터셋의 크기를 물리적으로 늘리는 것은 비효율적이기에, 원본 이미지를 줄여서 메모리 자체를 줄이면 되지 않을까? 하는 생각으로 데이터 활용 효율을 높여보려 했습니다.
- 그러나 원본 이미지를 사전 축소하는 것은 과정의 공정에 있어서 비효율적인데다 데이터 품질 손실을 초래하는 문제가 있었습니다.

### 해결 방안
- **데이터 증강(Data Augmentation) 도입:** **`VQAMCDataset` 클래스 내**에 `torchvision.transforms` 기반의 **무작위 변형 로직** (좌우 반전, 색상 변조 등)을 추가하여, 원본 이미지의 품질 손실 없이 학습 데이터의 다양성을 획기적으로 높여 모델의 **일반화 성능을 향상**시키는 전략을 채택했습니다.

### 데이터 증강
`VQAMCDataset` 클래스에서 `train` 데이터셋에 한해 다음과 같은 증강 기법을 적용하여 모델의 강건함(Robustness)을 높였습니다.

1. `ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2, hue=0.1)`: 이미지의 밝기, 대비, 채도, 색조를 무작위로 변경하여 다양한 조명 환경에 모델이 적응하도록 합니다. 이때 너무 큰 값은 의미를 왜곡시킵니다. 특히 색깔 편향(color bias) 값을 더욱 줄여서 색깔 관련 질문이 아닐 때의 일반화 성능을 높이고 밝기나 색감이 달라도 같은 사물로 인식하도록 했습니다.
2. `RandomHorizontalFlip()`: 이미지를 50% 확률로 좌우 반전시킵니다. (단, "왼쪽/오른쪽"을 묻는 질문이 많은 경우 제외해야 함)
3. `ToTensor()` 및 `Normalize()`: 이미지를 텐서로 변환하고, 모델이 사전 학습될 때 사용된 평균(mean)과 표준편차(std)를 정규화합니다.

## 3. 시행착오: 중-후기 및 해결
VQA 모델의 성능을 끌어올리기 위해 시도했던 주요 튜닝 과정으로 **더 큰 모델**, **더 높은 해상도**, 그리고 **과적합 제어**를 진행하였습니다.

## 3-1. 베이스 모델 변경 (Qwen2.5-VL-7B-Instruct)

초기 모델보다 더 강력하고 VQA 태스크에 특화된 모델을 사용했습니다.

`MODEL_ID = "Qwen/Qwen2.5-VL-7B-Instruct"`

이유:

- **Vision-Language (VL) 특화:** 이름에서 알 수 있듯이 이미지와 텍스트를 함께 이해하도록 설계된 최신 VLM(Vision-Language Model)입니다.
- **7B의 강력한 성능:** 70억 개의 파라미터를 가진 모델로, 긴 문맥도 잘 유지하면서 더 복잡한 추론과 이미지 이해가 가능합니다.

| 비교 기준 | **Qwen 2.5–3B** | **Qwen 2.5–7B**  | **Qwen 2.5–72B** |
| --- | --- | --- | --- |
| 문맥 이해 | 짧은 문장 위주 | 긴 문맥도 잘 유지 | 복잡한 추론 가능 |
| 수학·코딩 | 기초 계산 | 대부분의 코딩 과제 가능 | 복잡한 논리·알고리즘까지 |
| VQA (이미지+질문) | 단순 매칭 위주 | 객체·문맥 분석 가능 | 장면 이해·추론까지 가능 |
| 한국어/중국어 이해 | 기본 수준 | 매우 안정적 | 거의 인간 수준 |
- **Instruct 튜닝:** "A, B, C, D 중 정답을 고르시오"와 같은 지시(Instruction)를 따르도록 미세 조정되어 있어, 현재 태스크의 프롬프트 형식에 완벽하게 부합합니다.

## 3-2. LoRa 하이퍼파라미터 설정

7B 모델을 A100 GPU에서 효율적으로 미세 조정하기 위해 LoRa(Low-Rank Adaptation)를 사용했으며, 파라미터는 다음과 같이 설정했습니다.

- `r = 64`
- `lora_alpha = 128`
- `lora_dropout = 0.05`

이유:

- **`r = 64` (Rank):** 랭크(`r`)는 LoRa 레이어의 학습 가능한 파라미터 수를 결정합니다. 7B 같은 대형 모델을 튜닝할 때는 `r` 값을 8이나 16보다 큰 64 정도로 설정하여, 모델이 태스크에 대해 더 풍부하고 복잡한 표현을 학습할 수 있도록 했습니다.
- **`lora_alpha = 128` (Scaling):** `alpha`는 LoRa 가중치의 스케일링 값입니다. 일반적으로 `r`의 2배(`alpha = 2 * r`)로 설정하는 것이 '국룰'처럼 사용되며, 이는 랭크를 높인 효과를 더 강조하여 학습이 안정적으로 이루어지도록 돕습니다.
- **`lora_dropout = 0.05`:** LoRa 레이어에도 Dropout을 적용하여 어댑터 자체가 과적합되는 것을 방지하고 일반화 성능을 높였습니다.

## 3-3. 이미지 해상도 증가 (384 → 448)

`IMAGE_SIZE = 448`

이유:

- **모델의 Pre-training 해상도:** Qwen-VL 모델은 `448x448` 해상도의 이미지로 사전 학습되었습니다.  즉, 모델의 내부 가중치와 필터들은 `448x448` **이미지의 공간적 특징 분포**에 최적화되어 있습니다. 미세 조정(fine-tuning) 시에도 이와 동일한 해상도를 사용하면, 모델의 Vision Encoder가 잠재력을 최대한 발휘할 수 있습니다.
- 또한 448해상도는 224보다 정밀하게 정보를 보존할 수 있고, 해상도가 높은 이미지에 비해서(ex/2000px이상의 고해상도 이미지) GPU 메모리 효율과 속도 또한 효율적입니다.
- **세밀한 정보 포착:** VQA 태스크는 이미지 속의 작은 글씨나 미세한 객체의 차이를 구분해야 할 수 있습니다. 해상도를 384에서 448로 높이면 모델이 더 선명한 이미지에서 세부 정보를 포착할 수 있어 정답률 향상에 큰 도움이 됩니다.

## 3-4. 과적합(Overfitting) 제어: Epoch 5 → 3 → 2 순으로 정확도가 상승함

이전 실험에서 Epoch 5로 학습을 완료했을 때, 훈련(train) 손실(loss)은 `0.0009`까지 떨어졌으나 실제 제출 점수는 **0.932**에 그쳤습니다.

- **문제 진단:** 이는 전형적인 **과적합**(Overfitting)입니다. 모델이 훈련 데이터를 너무 완벽하게 외운 나머지, 새로운 테스트 데이터에 대한 일반화 성능이 오히려 떨어진 것입니다.
- **해결책:** 학습 로그를 검토한 결과, Epoch 3 부근에서 손실이 충분히 낮아지고 있었습니다. (loss=`0.0016`)
- **최종 전략:** Epoch 5까지 학습하는 대신 **Epoch 3, Epoch 2**로 낮춰보는 등 학습을 조기 종료(Early Stopping)하는 전략을 선택했습니다. 이를 통해 모델이 훈련 데이터의 잡음(noise)까지 외우기 직전, 즉 일반화 성능이 가장 좋은 시점의 모델을 확보하여 0.95 이상의 점수를 목표로 했습니다.


요약하자면, **(1) VQA에 특화된 강력한 7B 모델**을 **(2) 기존 학습에 맞는 해상도의 이미지**와 **(3) 적절한 데이터 증강**으로 학습시키되, **(4) 2 Epochs만 학습**시켜 과적합을 피하는 것이 핵심 전략이었습니다.

## 4. 팀원 소감

### 👩‍💻 이승연

AI 이론만 배워놓고 어떻게 모델을 구상할지 막막했는데, 팀원들과 함께 꾸준하게 데이터 증강을 진행하고 하이퍼파라미터를 변경하다 보니 목표에 가까운 정확도의 모델을 만들어낼 수 있어서 뿌듯했습니다. 특히 팀장인 윤명진 팀장님께서 주말에도 모델 자체를 바꾸고, 우리가 흔히 알고있는 epoch 수를 늘리면 정확도가 높아질 것이라는 생각을 깨고 epoch 수를 오히려 줄여가며 과적합을 방지한 것이 저희 팀의 가장 큰 장점이자 저희 팀의 전략의 핵심이었다고 생각합니다. 앞으로도 이런 팀 차원의 챌린지가 많이 만들어졌으면 좋겠습니다. 

### 👨‍💻 이승민

AI 처음엔 막연하게만 생각했는데 이번 이미지 분류 프로젝트를 하면서 활용도를 좀 더 알게된 느낌입니다.
솔직히 처음엔 개념 잡기가 쉽지 않았는데, 곁에서 든든하게 이끌어주고 도와준 팀원들 덕분에 정말 큰 힘이 됐습니다.

특히 학습 횟수(Epoch)나 여러 파라미터 값을 조절할 때마다 모델의 정확도가 눈앞에서 실시간으로 달라지는 걸 보는 과정이 정말 신기하고 인상 깊었습니다.
이 경험을 통해, 데이터를 '가장 잘' 설명할 수 있는 최적의 파라미터 값을 찾는 과정이 AI 모델 성능의 핵심이라는 것을 확실히 깨닫게 되었습니다.
